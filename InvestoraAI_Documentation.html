Project Overview

InvestoraAI is a full-stack AI-powered financial intelligence platform. A LangGraph ReAct agent runs weekly automated stock analysis — fetching market prices, fundamentals, and news — then computes quality + momentum scores and delivers a structured report to a React dashboard with user authentication, watchlists, price alerts, and Telegram notifications.

Core Features

📊 Weekly Analysis Engine

LangGraph ReAct agent fetches market, fundamental, and news data for 10 tickers, computes composite scores, synthesizes insights via GPT-4o-mini.

👤 User Accounts

Full registration and login system with user profiles (risk tolerance, interests, market preferences, currency, notification settings).

📱 Telegram Alerts

High-signal stocks (score ≥ 8.5 or ≤ 1.5) trigger real-time Telegram notifications via n8n Cloud to @InvestoraAgent_bot.

🤖 Chat Assistant

Floating chat widget on the dashboard with 25 predefined Q&A covering platform usage, investment basics, account settings, and more.

⭐ Watchlists & Alerts

Users can maintain custom watchlists and configure price alerts (above/below/daily move thresholds) for individual stocks.

📈 Individual Stock Analysis

Dedicated stock pages with price charts, recent news, AI synthesis, and historical signal tracking per ticker.

Stock Universe (Default — 10 tickers)

AAPL · AMZN · GOOGL · JPM · MA · META · MSFT · NVDA · TSLA · V

Scoring Model

Momentum Score0.455-day return, volume trend, news sentiment0–10 → Strong / Positive / Neutral / Negative
Composite Score—Quality × 0.55 + Momentum × 0.450–10 final ranking

UserContext

The UserContext.tsx provides app-wide authentication state via the useUser() hook. It manages: session rehydration from localStorage, login/logout, profile updates, and watchlist sync from the backend (GET /user/:userId/watchlist). All components use const {'{ user, logout }'} = useUser().

Authentication System

Architecture

  Browser                      n8n Cloud                    Static Data
  ───────                      ─────────                    ───────────
  username + password
       │
       ▼ SHA-256 hash (client-side, never sends plaintext)
  { username, passwordHash }
       │
       ▼ POST
  /webhook/investora/register ──► Code node ──► store user ──► { users: {...} }
  /webhook/investora/login    ──► Code node ──► verify     ──► return profile
  /webhook/investora/profile  ──► Code node ──► update     ──► save changes
       │
       ▼
  localStorage: { username, userId, profile }
  (session rehydrated on refresh via UserContext)

UserProfile Interface (v2)

interface UserProfile {
  // Core fields
  riskTolerance: "low" | "medium" | "high";
  interests: string[];            // tech, crypto, energy, forex, commodities
  telegramChatId: string;

  // v2 fields (optional for backward-compatibility)
  displayName?: string;
  riskTolerancePercent?: number;  // 0-100 slider → maps to low/medium/high
  defaultMarket?: "US" | "EU";
  baseCurrency?: "USD" | "EUR" | "GBP";
  dailyEmailDigest?: boolean;
  alertNotifications?: boolean;
}

n8n Auth Endpoints

EndpointMethodPayloadResponse
/webhook/investora/registerPOST{username, passwordHash, userId, profile}{ok, userId}
/webhook/investora/loginPOST{username, passwordHash}{ok, userId, profile}
/webhook/investora/profilePOST{username, profile}{ok}

Security Notes

⚠️
Passwords are SHA-256 hashed client-side with salt investora_2026 before being sent to n8n. The plaintext password never leaves the browser. User records are stored in n8n workflow static data (persists between executions).

Graph Execution Flow

  init_state
       │  set run_id, date, ticker list
       ▼
  plan_next_action  ◄─────────────────────────────────────────────────┐
       │  GPT-4o-mini decides which tool to call for which ticker          │
       ▼                                                                    │
  execute_tool_action  (Marketstack / FMP / Finnhub / yfinance / RAG)    │
       │  caches results to data/cache/                                    │
       └──────────────────────────────────────────────────────────────────►┘
       │  (when all tickers + tools processed)
       ▼
  compute_scores  (Quality×0.55 + Momentum×0.45)
       │  ROE, margins, debt ratios, EPS growth, 5-day return
       ▼
  synthesize_evidence  (GPT-4o-mini)
       │  → ranked opportunities, quality/momentum narratives, risk factors
       ▼
  emit_signals  (classify ALERT / WEEKLY_CANDIDATE / MONITOR)
       │
       ├──► post_alerts      (score ≥8.5 or ≤1.5 → n8n Telegram Alerts)
       ├──► post_candidates  (7.0–8.4 → n8n Weekly Candidates Email)
       │
       ▼
  assemble_report_json  → ReportPayload TypedDict
       │
       ▼
  assemble_markdown  → human-readable version
       │
       ▼
  post_to_n8n  → full report to N8N_WEBHOOK_URL
       │
       ▼
  persist_report  → data/reports/YYYY-MM-DD.json
       │
       ▼
  persist_snapshot  → SQLite + MONITOR_WEBHOOK_URL
       │
       ▼
   DONE  →  SSE: {"type":"done","run_id":"..."}

Backend API Reference

Base URL: https://investoraai.onrender.com

GET  /latest-report

Returns the most recent weekly report JSON from data/reports/. Frontend caches for 1 hour via React Query.

POST  /run-analysis-stream  text/event-stream

Runs full LangGraph analysis and streams SSE progress events. Used by dashboard "Generate Report" button.

Body: { "tickers": ["AAPL"], "skip_synthesis": false, "no_post": false }
SSE:  node_complete · done · error · ping
POST  /run-weekly  🔒 CRON_SECRET

Synchronous full run for cron jobs. Requires Authorization: Bearer <CRON_SECRET> when CRON_SECRET is set.

GET  /run-history?limit=10

Returns last N analysis run metadata from SQLite (run_id, date, error_count, signal_count, failed_tickers).

GET  /ticker-history/{ticker}?weeks=6

Returns signal history for a specific ticker over the last N weeks (max 52) from SQLite.

Known Limitations & Roadmap

Future Roadmap

🔐 Move Auth to FastAPI

Migrate register/login/profile from n8n static data to the existing FastAPI backend with SQLite. Free, reliable, no n8n dependency for auth.

☁️ Self-host n8n on Render

Deploy n8n as Docker container on Render (~$7/mo). Enables $env variables, persistent storage, no trial limitations.

💾 Persistent Storage

Add Render Disk or migrate SQLite to PostgreSQL for reliable run history and user data across restarts.

🤖 AI Chat Upgrade

Upgrade chat agent from predefined Q&A to real OpenAI-powered conversations via n8n "investora-chat-agent" workflow.

⏰ Scheduled Analysis

n8n cron workflows already configured — activate Weekly Full Run (Monday 06:00 UTC) and Daily Fast Scan.

🔒 Regenerate Bot Token

Use BotFather /revoke to generate a new Telegram bot token and update in Render + n8n credentials for security.
